# 2026-05-01-005 阶段 B MCP 能力边界实测

## 背景

用户完成步骤 1-4（创建 AttributeSet 代码 + 编译），要求 MCP 创建 GE_Hero_AttributeInit 和 GE_Damage_Test 两个 GameplayEffect 蓝图。实测 MCP 当前能力的可用与不可用边界。

## 测试环境

- UE 5.3 Editor 已启动，MCP Bridge 9876 端口就绪
- 项目已编译（包含 UUMCombatAttributeSet C++ 类）
- 测试方法：PowerShell TCP 直连 → execute_python_snippet + list_assets + get_asset_info

## 逐项测试结果

### 1. 创建目录
```python
unreal.EditorAssetLibrary.make_directory("/Game/GAS/GameplayEffects/")
```
✓ 成功。list_assets 确认目录存在。

### 2. 创建 GE 蓝图（非 Actor 父类）
```python
factory = unreal.BlueprintFactory()
factory.set_editor_property("parent_class", unreal.GameplayEffect)
bp = asset_tools.create_asset("GE_Hero_AttributeInit", "/Game/GAS/GameplayEffects/", unreal.Blueprint, factory)
```
✓ 成功。对比 `create_blueprint` Handler 的 `IsChildOf(AActor)` 限制，Python 绕过了这个检查。

### 3. 设置 DurationPolicy
```python
cdo = unreal.get_default_object(bp.generated_class())
cdo.set_editor_property("duration_policy", unreal.GameplayEffectDurationType.INSTANT)
```
✓ 成功。验证读取返回 `INSTANT`。

### 4. 查找 C++ 类
| 方法 | 路径 | 结果 |
|------|------|------|
| `unreal.find_class` | — | 不存在 |
| `unreal.load_class` | `/Script/UnrealEditorMCP.UClass` | ✗ |
| `unreal.find_object` | `/Script/UnrealEditorMCP.UMCombatAttributeSet` | ✓ (UClass) |
| `unreal.find_object` | `/Script/UnrealEditorMCP.Default__UMCombatAttributeSet` | ✓ (CDO) |

**发现**：引擎反射路径中类名不带 U 前缀（`UMCombatAttributeSet` 非 `UUMCombatAttributeSet`）。

### 5. 获取 FProperty（Health 属性）
| 方法 | 结果 |
|------|------|
| `find_object(None, "/Script/.../UMCombatAttributeSet:Health")` | ✗ null |
| `UClass.get_editor_property("Health")` | ✗ 错误 |
| `dir(UClass)` → 无 `get_property` / `find_property_by_name` | ✗ |

**结论**：UE 5.3 Python API 不暴露 UClass → FProperty 的查询路径。无法从 Python 构造 `FGameplayAttribute`。

### 6. GameplayModifierInfo 创建
```python
mod = unreal.GameplayModifierInfo()  # ✓ 可实例化
```
但无法设置其 `Attribute` 字段（需要 FGameplayAttribute），也无法设置 `ModifierMagnitude` 内部结构。

### 7. GameplayTag 创建
无可用的 MCP Handler。Python 路径需要直接操作 `UGameplayTagsManager`，风险高，未尝试。

### 8. Python output 捕获
`MCPPythonHandler.cpp:34` 硬编码 `SetStringField(TEXT("output"), TEXT(""))`。Python `print()` 和 `unreal.log()` 均不捕获。本次通过写入磁盘文件绕过。

## 最终结论

| 可 MCP 做 | 需用户手动 |
|-----------|-----------|
| 创建 GE 蓝图资产 | 添加 Modifier（Attribute + Op + Magnitude） |
| 设置 DurationPolicy | 创建 GameplayTag |
| 删除资产 | 配置 SetByCaller Magnitude |
| 验证资产存在 | 应用初始 GE 的步骤 6 代码 |
