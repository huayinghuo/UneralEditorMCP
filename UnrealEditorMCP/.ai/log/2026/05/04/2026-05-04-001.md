## 背景

用户基于上次实测的 MCP 能力缺口，新增了 CDO 属性编辑器（4 Handler）并修复了 `create_blueprint` 的 Actor 限制、GameplayTag 持久化、Python 输出捕获。本轮测试使用 MCP 完整走通 GE 创建→配置全流程。

## 新增 MCP 能力

| Handler | action 名 | 用途 |
|---------|----------|------|
| `FMCPBlueprintGetCDOPropertyHandler` | `blueprint_get_cdo_property` | 读取 CDO 任意属性 |
| `FMCPBlueprintSetCDOPropertyHandler` | `blueprint_set_cdo_property` | 设置 CDO 标量/枚举属性 |
| `FMCPBlueprintAddCDOArrayHandler` | `blueprint_add_cdo_array` | 向 CDO TArray 追加元素（支持 ImportText） |
| `FMCPBlueprintRemoveCDOArrayHandler` | `blueprint_remove_cdo_array` | 按 index 删除 TArray 元素 |
| `FMCPCloseModalWindowHandler` | `close_modal_window` | 关闭编辑器模态弹窗 |

### 修复项
- `create_blueprint`：`IsChildOf(AActor)` → `IsChildOf(UObject)` + `NotBlueprintable` 检查
- `create_gameplay_tag`：`GConfig->Flush` → `FFileHelper` + `FConfigFile` 直接文件 IO
- `execute_python_snippet`：`ExecPythonCommand` → `ExecPythonCommandEx` + `log` 输出

## ImportText 格式参考

```
# 标量值 Modifier
(Attribute=(AttributeName=PropertyName),ModifierOp=Override,ModifierMagnitude=(MagnitudeCalculationType=ScalableFloat,ScalableFloatMagnitude=(Value=NUMBER)))

# SetByCaller Modifier  
(Attribute=(AttributeName=PropertyName),ModifierOp=Additive,ModifierMagnitude=(MagnitudeCalculationType=SetByCaller,SetByCallerMagnitude=(DataName=TagName)))
```

ModifierOp 枚举值: `Additive`, `Multiply`, `Divide`, `Override`
