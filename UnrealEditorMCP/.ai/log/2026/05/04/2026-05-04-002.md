## 修复 1: blueprint_add_cdo_array field_overrides

### 文件
`Plugins/UnrealEditorMCPBridge/Source/UnrealEditorMCPBridge/Private/Handlers/MCPBlueprintUtilityHandlers.cpp`

### 变更点

#### A. `FFieldPathProperty::SetObjectPropertyValue` 不存在（编译错误）
- **位置**: Line 289-290
- **原代码**: `FieldPathProp->SetObjectPropertyValue(FieldAddr, RefProp);`
- **根因**: UE 5.3 的 `FFieldPathProperty` 没有 `SetObjectPropertyValue` 方法（FFieldPath 包装 FField*，非 UObject*）
- **修复**: `*static_cast<FFieldPath*>(FieldAddr) = FFieldPath(RefProp);` — 直接写内存，等效于 `TTypeFundamentals::SetPropertyValue`
- **验证**: `TFieldPath<T> : public FFieldPath` (POD), 直接写入 `FFieldPath` 子对象安全

#### B. `FindFirstObject<FProperty>` 不可用（编译错误）
- **位置**: Line 275-278
- **原代码**: `FindFirstObject<FProperty>(*(ClassPath + TEXT(":") + PropName), ...)`
- **根因**: `FProperty` 继承自 `FField`（非 UObject），`FindFirstObject` 模板要求 `UObject` 子类
- **修复**: `FindFirstObject<UClass>(*ClassPath, ...)` + `RefClass->FindPropertyByName(FName(*PropName))`

#### C. `FGameplayAttribute` 内部 `AttributeName` 未同步
- **位置**: Line 291-309（FStructProperty 分支内）
- **原代码**: 仅设置 FFieldPath，未处理 FStrProperty
- **根因**: `FGameplayAttribute` 有 `FString AttributeName` 字段，仅设 `TFieldPath<FProperty> Attribute` 时 `GetName()` fallback 可工作，但持久化需要显式写入
- **修复**: 追加 `FStrProperty` 分支，当 `InnerStr->GetName() == TEXT("AttributeName")` 时写入 `RefProp->GetName()`

### 测试
```
action: blueprint_add_cdo_array
payload: {
  asset_path: "/Game/GAS/GameplayEffects/GE_Hero_AttributeInit",
  property_name: "Modifiers",
  value: "(ModifierOp=Override,...)",
  field_overrides: {"Attribute": "/Script/UnrealEditorMCP.UMCombatAttributeSet:Health"}
}
```
结果: `Attribute=(AttributeName="Health",Attribute=/Script/UnrealEditorMCP.UMCombatAttributeSet:Health)` ✅

## 修复 2: create_gameplay_tag 持久化

### 文件
`Plugins/UnrealEditorMCPBridge/Source/UnrealEditorMCPBridge/Private/Handlers/MCPGameplayTagHandlers.cpp`

### 变更点

#### A. `AddNativeGameplayTag` 运行时无操作
- **根因**: `DoneAddingNativeTags()` 在引擎初始化后调用，此后 `AddNativeGameplayTag` 为 no-op
- **修复**: 直接修改 `GetMutableDefault<UGameplayTagsSettings>()->GameplayTagList`

#### B. 持久化方式错误
- **原代码**: `GConfig->SetArray` / `GConfig->Flush` / `FFileHelper::SaveStringToFile`
- **根因**: GConfig 不认未注册的 `DefaultGameplayTags.ini`；文件写入方案被后续 CDO 覆盖
- **修复**: `Settings->TryUpdateDefaultConfigFile(ConfigPath)` — 直接通过 Settings CDO 写默认配置文件

#### C. 数据结构格式错误
- **原代码**: 写入纯字符串 `"TagName"`
- **根因**: `GameplayTagList` 类型是 `TArray<FGameplayTagTableRow>`，每个元素需 `(Tag="...",DevComment="...")` 格式
- **修复**: 构造 `FGameplayTagTableRow(FName(*TagName), Desc)` 并 `AddUnique` 到 CDO 数组

#### D. 运行时 tag 树未重建
- **根因**: 仅修改 CDO 但 `GameplayTagNodeMap` 未更新，`RequestGameplayTag` 查询结果不变
- **修复**: 调用 `Mgr.EditorRefreshGameplayTagTree()` → `DestroyGameplayTagTree()` → `LoadGameplayTagTables()` → `ConstructGameplayTagTree()` 重建整个 tag 树

### 新增 include
```cpp
#include "GameplayTagsSettings.h"
```

### 测试

```
payload: { tag_name: "State.Stunned", description: "Stunned" }
```
结果: `valid=True, created=True, tag=State.Stunned` ✅
追加调用返回 `created=False` ✅
`DefaultGameplayTags.ini` 持久化成功 ✅
