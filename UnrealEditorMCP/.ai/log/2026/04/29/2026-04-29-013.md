## 4 项修复

### 1. execute_python_snippet 失败语义修正

**原行为**：Python 执行失败时返回 `ok=true, result.success=false`
**修复后**：返回 `ok=false, error.code=PYTHON_EXEC_FAILED`

变更文件：`MCPPythonHandler.cpp`
```cpp
// 旧：return true（始终）
// 新：return false + PYTHON_EXEC_FAILED
if (!bSuccess) {
    MCPBridgeHelpers::BuildErrorResponse(TEXT("PYTHON_EXEC_FAILED"), ...);
    return false;
}
```

### 2. 资产创建冲突检测

**原行为**：create_blueprint / create_widget_blueprint 不检查重名，可能导致覆盖
**修复后**：通过 AssetRegistry 检测目标路径是否已有资产

变更文件：`MCPBlueprintHandlers.cpp`, `MCPWidgetHandlers.cpp`
```cpp
IAssetRegistry& AR = ...;
FAssetData Existing = AR.GetAssetByObjectPath(FSoftObjectPath(ObjectPath));
if (Existing.IsValid()) → ASSET_CONFLICT 错误
```

**编译问题**：初次用 FindObject/FindPackage 导致 CRASH → 改用 AssetRegistry

### 3. 断连时自动结束事务

**原行为**：客户端断开后活跃事务悬挂，编辑器处于不确定编辑状态
**修复后**：Run() 检测到客户端断开时调用 `MCPTransaction_AutoEndIfActive()`

变更文件：
- `MCPTransactionHandlers.h` — 声明 `void MCPTransaction_AutoEndIfActive()`
- `MCPTransactionHandlers.cpp` — 实现：检查 GEditor + bTransactionActive，调用 EndTransaction
- `MCPBridgeServer.cpp` — 断连处理中调用该函数

### 4. get_mcp_config 增强

**原行为**：只返回 action 列表
**修复后**：新增 `mode`（development/restricted）、`version`、`ue_version`、`bridge_protocol` 字段

变更文件：`MCPQueryHandlers.cpp`, `MCPBridgeServer.h`
