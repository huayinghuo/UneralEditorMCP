## 调试日志

### 第 0 轮：FTSTicker + 非阻塞 socket
**现象**：客户端连接成功，但所有请求超时无响应
**UE 日志**：
```
LogMCPBridgeServer: MCP Bridge listening on 127.0.0.1:9876
LogMCPBridgeServer: Client connected from 127.0.0.1:59381
```
（仅此两条，无 Queuing/Processing/Sending 日志）
**尝试修复 1**：FTSTicker → FCoreDelegates::OnBeginFrame
**结果**：依然无响应，OnBeginFrame 也未触发 ProcessPendingRequests

### 第 1 轮：添加诊断日志
**修改**：在 ProcessPendingRequests、HandleRequest、SendResponse、数据接收处添加 UE_LOG
**结果**：仍然无诊断日志输出，确认数据根本未进入处理管道

### 第 2 轮：多次启动导致端口占用
**现象**：`Failed to bind to 127.0.0.1:9876`
**根因 1**：`Stop-Process` 未完全杀掉前一 Editor 进程（多个 PID 累积）
**根因 2**：前一进程端口处于 TIME_WAIT 状态
**修复**：
- 启动前彻底 kill 所有 UnrealEditor 进程
- `ListenerSocket->SetReuseAddr(true)` 允许端口重用

### 第 3 轮：切换到阻塞 socket（核心修复）
**根因分析**：非阻塞 socket 的 `Recv` 在数据到达时未正确返回。UE 5.3 中 `FSocket::Recv` 对非阻塞 socket 在某些情况下即使数据已到达也可能返回 0 字节（行为取决于底层实现）
**修复**：
- 移除 `ClientSocket->SetNonBlocking(true)`
- 使用 `Wait(ESocketWaitConditions::WaitForRead, FTimespan::FromMilliseconds(50))` + 阻塞 Recv
- Accept 后不 skip 当前迭代（移除 continue），立即刷新 ActiveClient 快照

**验证结果**：
```
connected → sent → recv[0]: {"id":"t1","ok":true,"result":{"message":"pong"},"error":null}
```
✅ Ping 响应首次成功返回！

### 第 4 轮：Python 错误返回结构修复
**现象**：`execute_python_snippet("1/0")` 返回 `ok: false, error: {code:"", message:""}`
**根因**：Python 执行失败时 handler 返回 `false`，SendResponse 使用 error path 但 OutErrorCode/OutErrorMessage 为空
**修复**：`execute_python_snippet` 始终返回 `true`，将执行状态放入 `result.success` 字段

### 第 5 轮：插件依赖声明
**警告**：`Plugin 'UnrealEditorMCPBridge' does not list plugin 'PythonScriptPlugin' as a dependency`
**修复**：`.uplugin` 添加 `"Plugins": [{"Name": "PythonScriptPlugin", "Enabled": true}]`
