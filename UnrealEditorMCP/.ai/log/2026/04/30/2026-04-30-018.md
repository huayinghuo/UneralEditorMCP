## Action 1: blueprint_apply_spec (Write)

**输入**:
```json
{
  "spec": {
    "blueprint": {"name": "MyBP", "parent_class": "Actor"},
    "nodes": [
      {"id": "b", "type": "event", "event_name": "ReceiveBeginPlay"},
      {"id": "p", "type": "call_function", "function_name": "PrintString", "params": {"InString": "hello"}}
    ],
    "edges": [
      {"from_node": "b", "from_pin": "then", "to_node": "p", "to_pin": "execute"}
    ]
  }
}
```

**执行流程**:
1. ParseSpecFromJson → extract name / parent_class / nodes[] / edges[]
2. IsParentClassActorDerived 校验
3. AssetRegistry 冲突检测 → ASSET_CONFLICT
4. FKismetEditorUtilities::CreateBlueprint
5. 遍历 nodes: event → FindEventNodeByMemberName / CreateEventNode; call_function → FindFirstObject<UFunction> → NewObject<UK2Node_CallFunction> → SetFromFunction → 设置 pin defaults
6. 遍历 edges: TryConnectPins(guid_map[from], pin, guid_map[to], pin)
7. MarkBlueprintAsStructurallyModified + SavePackage
8. 返回 id_guid_mapping 供后续操作

## Action 2: blueprint_export_spec (Read)

**输出**:
```json
{
  "blueprint": {"name": "...", "asset_path": "...", "parent_class": "Actor"},
  "nodes": [{id, type, event_name/function_name, node_guid, pins[{name, direction, type, default}]}],
  "edges": [{from_node_guid, from_pin, to_node_guid, to_pin}],
  "node_count": 4, "edge_count": 1
}
```

**执行流程**:
1. LoadBlueprint → GetEventGraph
2. 遍历 Nodes: Cast UK2Node_Event → event_name; Cast UK2Node_CallFunction → function_name
3. 遍历 Pins: name / direction / type / default
4. 遍历 Connected pins: linked_to → edge
5. 生成稳定 ID（type_name_index 格式，支持 round-trip）

## 修复：round-trip 边格式兼容性

**问题**: `export_spec` 边使用 `from_node_guid`/`to_node_guid`（UE GUID），但 `apply_spec` 期望 `from_node`/`to_node`（spec ID），导致 export 结果无法直接作为 apply 输入。

**修复** (`MCPBlueprintSpecHandlers.cpp:213-230`):
1. 新增 `TMap<FGuid, FString> GuidToSpecId` 映射表
2. 导出节点时记录 `node_guid → spec_id` 映射
3. 导出边时查询映射表，输出 `from_node`/`to_node`（spec ID）非 GUID

**验证**: apply → export → re-apply 边格式一致（`event_ReceiveBeginPlay_0.then → call_PrintString_0.execute`）。
