## 10C 规划成果

### 三层配置模型

| 层次 | 来源 | connected | 触发条件 |
|------|------|-----------|----------|
| Bootstrap | `BOOTSTRAP_MCP_CONFIG` (Python 本地常量) | `False` | UE TCP 不可达 |
| Live | C++ `get_mcp_config` 返回 | `True` | UE TCP 可达 |
| Degradation | 切换回 Bootstrap | `False` | UE TCP 断连 |

### Bootstrap Config 结构

```python
BOOTSTRAP_MCP_CONFIG = {
    "version": "1.0.0",
    "bridge_protocol": "TCP/JSON Lines",
    "source": "python-bootstrap",
    "connected": False,
    "ue_version": None,
    "mode": "development",
    "token_enabled": False,
    "actions": [
        {"action": "get_mcp_config", "category": "Read", "mode": "development"},
    ],
    "count": 1,
}
```

### Live Override 逻辑

```python
# call_tool("ue_get_mcp_config") 在线路径
live_config = result["result"]
live_config["source"] = "ue-live"
```

### Disconnect Degradation 逻辑

```python
# list_tools() 离线路径
actions = BOOTSTRAP_MCP_CONFIG["actions"]
for entry in actions:
    schema = TOOL_SCHEMAS.get(entry["action"])
    tools.append(types.Tool(**schema))

# call_tool("ue_get_mcp_config") 离线路径
if tool_name == "ue_get_mcp_config":
    return BOOTSTRAP_MCP_CONFIG  # 直接返回 JSON
# 其他 tool 离线仍返回 ConnectionError
```

### 关键设计约束

1. `list_tools()` 离线结果与 `call_tool("ue_get_mcp_config")` 离线结果 **必须来自同一 `BOOTSTRAP_MCP_CONFIG`**
2. bootstrap 不伪装成 UE live registry（`source` 字段显式区分）
3. bootstrap 只在 `actions` 中包含 Python 本地已知的最小工具集合
4. 不引入持久缓存或磁盘快照机制
